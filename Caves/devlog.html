Devlog #4: The Architecture & Security Update

May 10th, 2026

As our player base grows, so do the challenges of running a persistent MMO! This week, we took a brief pause from adding new content to perform a massive "under-the-hood" rewrite. We've restructured our database to prevent server-meltdowns, bolstered our anti-cheat mechanics, and added some highly requested Quality of Life (QoL) features to the Stash.

Sys Serverless Map Migration

  • RTDB Integration: Dropped items, chopped trees, and broken walls are now synchronized using Firebase's Realtime Database instead of Firestore. This massively reduces database write quotas and allows changes to appear to other players with near-zero latency.
  • AI Heartbeat Throttling: Fixed an issue where 50+ players holding the 'W' key would essentially DDOS our own servers by constantly requesting to move the AI. The client now uses a strict 600ms gatekeeper to drastically reduce network calls.
  • Distributed Garbage Collection: The game now intelligently sweeps the map every 50 turns. If an enemy is completely abandoned and no players are within 100 tiles, it is safely de-spawned to save server memory.

Sec Anti-Cheat & Save Integrity

  • Robust Hashing: The Cloud Backup signature system has been upgraded. It now cryptographically hashes your maxHealth, inventory size, and stash size. This prevents players from manually injecting hundreds of Legendary items into their JSON save file before restoring it.
  • Legacy Support: Don't worry! If you have a backup from before this update, the game will automatically fall back to the old validation method so you don't lose your progress.
  • Hardware Acceleration: Game IDs (UUIDs) are now generated using your device's native crypto.randomUUID() chip when available, making generation 10x faster and mathematically secure.
