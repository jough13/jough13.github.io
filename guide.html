How to Create a New Blog Post

A detailed, step-by-step guide to adding content to your site.

Welcome to the user guide! This document explains how to add a new post to your blog. The system is designed to be simple, using two key parts for every post: a Markdown file for your content and an entry in the posts.json file to make it appear on the site.

Step 1: Crafting Your Content (The .md File)

First, you need to write the actual content of your post. We do this in a simple text file with the .md extension, which stands for Markdown. Markdown is a lightweight language for formatting text.

File Naming and Location

  • Create the file: Create a new file and save it with a descriptive name. Use lowercase letters, numbers, and hyphens. For example, a-guide-to-photosynthesis.md is better than A Guide to Photosynthesis!!.md.
  • Place the file: Place this file inside a relevant folder, such as /posts/, /science/, or /recipes/. Keeping things organized helps! The path you choose is important for Step 2.

Step 2: Registering Your Post (posts.json)

Your .md file contains the content, but the website doesn't know it exists yet. To make it appear, you must add an entry to the posts.json file. This file acts as a master list or "manifest" for all your posts.

The Structure of a Post Entry

Open posts.json and add a new JSON object to the top of the list (this ensures newest posts appear first). It must have the following structure:

{
  "title": "The Title of Your New Post",
  "date": "YYYY-MM-DD",
  "file": "path/to/your/markdown-file.md",
  "summary": "A short, one-or-two-sentence teaser that appears on the homepage.",
  "category": "the-main-category",
  "tags": ["tag1", "a-second-tag", "another"]
}

Field Explanations:

  • title: The full title of your post, which will be displayed in large text.
  • date: The publication date. Use the YYYY-MM-DD format for consistency.
  • file: This is crucial! It's the exact path to the .md file you created in Step 1. For example, "science/photosynthesis-deep-dive.md".
  • summary: A brief description shown on the main blog page. Keep it concise and engaging.
  • category: Choose one of the existing site categories: ideas, science, recipes, art, or links. This ensures consistent navigation and styling.
  • tags: A list of lowercase keywords. These are more specific than the category. For example, a post in recipes might have tags like italian, pasta, vegetarian.

Practical Example

Imagine this is the top of your posts.json file:

[
  {
    "title": "My First Blog Post",
    "date": "2025-09-21",
    "file": "posts/09212025.md",
    "summary": "This is a short teaser about the first post...",
    "category": "ideas",
    "tags": ["personal", "webdev", "meta"]
  }
]

To add a new post about lasagna, you would add it to the top:

[
  {
    "title": "The Perfect Lasagna Recipe",
    "date": "2025-09-22",
    "file": "recipes/perfect-lasagna.md",
    "summary": "Learn how to make the best lasagna of your life...",
    "category": "recipes",
    "tags": ["food", "italian", "cooking"]
  },
  {
    "title": "My First Blog Post",
    "date": "2025-09-21",
    "file": "posts/09212025.md",
    "summary": "This is a short teaser about the first post...",
    "category": "ideas",
    "tags": ["personal", "webdev", "meta"]
  }
]

Notice the comma , added after the closing brace } of the new entry, separating it from the next one.

Troubleshooting & Tips

My post isn't showing up!

  • Check the file path: This is the most common error. Double-check the path in posts.json to make sure it exactly matches the folder and filename of your .md file.
  • Check JSON syntax: A missing comma between post entries, or an extra comma after the very last entry, will break the file. You can use an online "JSON validator" to check your file for errors.

Images in my post are broken.

  • Image paths in your Markdown file are relative to the location of the .md file itself. If your post is at science/my-post.md and your image is at assets/images/diagram.png, the Markdown path would likely be ../assets/images/diagram.png (the ../ means "go up one directory").
