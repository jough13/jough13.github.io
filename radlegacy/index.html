User Manual & Data Guidelines

1. Uploading Data

The Navigator accepts `.xlsx`, `.xls`, and `.csv` files. You can drag and drop multiple files at once. If you upload an Excel workbook with multiple tabs, the engine will automatically read every single tab.

To restore a previously saved state (including your custom field notes), upload a `.json` backup file generated by this tool.

2. Data Structure & Fuzzy Matching

The tool uses a "Fuzzy Parsing" engine to read your spreadsheets. It looks for specific column headers but will gracefully adapt to variations. To ensure perfect data mapping, use the following column headers in your files:

  • Major Location/Command (Fallbacks: Command, Location. If missing, defaults to the Excel Tab Name)
  • Sub-Location/Site Name (Fallbacks: Site Name, Site)
  • Site Identifier/Number/Description (Fallbacks: Description, Site Description)
  • Radiological Classificiation (Fallbacks: Radiological Classification, Status, Class)
  • ROPCs (Fallbacks: Nuclides, Isotopes, Radionuclides)
  • Justificiation (Fallbacks: Justification)
  • Current Land Use/Restrictions (Fallbacks: Land Use, Restrictions)
  • Comments/Notes (Fallbacks: Comments, Notes)
  • HRA Report Title and HRA Report Issue Date

3. Data Normalization Engines

Rad Status: The tool scans your classification column and standardizes it into one of six core categories: Impacted, Potentially Impacted, Area of Interest, Non-Impacted, Previously Remediated, or Other.

Nuclides: The ROPC engine reads human-entered strings (e.g., "Depleted Uranium", "U-238", "Cesium") and translates them into uniform tags (e.g., "DU", "Cs-137") for accurate filtering.
